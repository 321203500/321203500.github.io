flask学习
2019-01-24 Read

1.虚拟环境

创建: virtualenv--no-site-packages -p xxxx freshenv8
安装flask: pip install flask
安装文件: pip install -r requirement.txt

2.Flask最小应用

2.1最小7行代码

#生成flask对象，传入__name__参数，表示模块名或包名
app = Flask(__name__)
        # 告诉服务器，访问127.0.0.1:5000/hello/地址可触发hello()方法
# 类似与Django中的urls.py文件的功能
@app.route('/hello/')
def hello():
    return 'hello world'
        # sys.argv接收启动命令时的参数，如: python hello.py 0.0.0.0 80
ip = sys.argv[1]
port = sys.argv[2]
# host代表启动的ip地址
# port代表启动的端口
# debug代表启动的模式，调试模型
app.run(host=ip, port=port, debug=True)

3.flask_script

使用: 用于管理flask启动的命令
安装: pip install flask_script
使用:
from flask_script import Manager
manage = Manager(flask应用)
启动: manage.run()
启动命令: python xxx.py runserver -p 端口 -h IP地址 -d 调试模式

4.flask-blueprint

背景: 模块化管理路由地址

安装: pip install flask-blueprint

4.1使用: 分两步

第一步: 管理路由
初始化: blue = Blueprint('first', __name__)
管理路由: @blue.route('路由')

第二步: 注册蓝图
flask对象.register_blueprint(blueprint=blue, url_prefix='/xxx')

重定向
直接重定向
redirect('/app/hello/'): 直接重定向到某个地址

反向解析重定向

有参情况
redirect(url_for('生成蓝图的第一个参数.重定向的函数名, 参数=值'))
无参情况
redirect(url_for('生成蓝图的第一个参数.重定向的函数名'))

5.路由规则

格式: <转换器：参数>
<int:id> 表示接受的id值为int类型
<string:name>表示接收的name值为字符串类型，默认可简写为<name>
<float: price>表示接收的price值为浮点数类型
<uuid:uid> 表示接收的uid值为uuid类型的参数
<path:p> 表示接收的p参数为path路径

6.错误抛出与捕获

导包: from flask import abort
使用: abort(500)
捕获: @blue.errorhandler(500)
抛出错误码时，错误码必须符合HTTP定义的状态码,且抛出的错误码要符合运用场景

7.请求与响应

7.1请求

接收参数
GET请求: request.args.get(key)或request.args.getlist(key)
POST/PUT/PATCH/DELETE请求: request.form.get(key)或request.form.getlist(key)
获取请求方式
request.method
获取路由
request.path
获取cookie
request.cookies
获取文件
request.files

7.2响应

创建: make_response(响应内容, 响应状态码)， 状态码默认为200，可不写。
响应对象.set_cookie(key, value, max_age)
响应对象.delete_cookie(key)

8.session

8.1思路

第一次访问任何一个路由地址时,在客户端，将在cookie中存储一个键值对，键为session，值为唯一的uuid类型的参数
第一次方式的同时，在服务端指定的redis中，也将存储cookie中的uuid值
如果用户登录成功，则可以像redis中存储的uuid参数中保存信息

8.2flask自带形式

存储数据: 将session中存储的键值对内容存放在cookie中
存储方式: 设置secret_key参数，越复杂越好
使用方式: 
from flask import session
session['key'] = value

8.3使用flask-session第三方库

安装: pip install flask-session
配置存储的数据库:
flask对象.config['SESSION_TYPE'] = 'redis'
flask对象.config['SESSION_REDIS'] = redis.Redis(host, port, password)
初始化:
from flask_session import Session
第一种: Session(flask对象)
第二种:
se = Session()
se.init_app(flask对象)

9.数据库

安装: pip install flask-sqlalchemy
创建对象:
from flask_sqlalchemy import SQLALChemy
db = SQLALChemy()

9.1模型定义:

class User(db.Model):
    # 定义自增的主键id
    id = db.Column(db.Integer, primary_key=True, autoincrement=True)
    # 定义长度为10的name字段，类型为字符串，唯一且不能为空
    name = db.Column(db.String(10), unique=True, nullable=False)
    # 定义密码password字段
    password = db.Column(db.String(249), nullable=False)

    __tablename__ = 'flask_user'

9.2配置链接:

flask对象.config['SQLALCHEMY_DATABASE_URI']='mysql+pymysql://root:123456@127.0.0.1:3306/flask8'

9.3初始化:

db.init_app(flask对象)

9.4模型的映射

进入交互环境: python manage.py shell 
导入模型: from app.models import db 
创建: db.create_all()
删除: db.drop_all()

9.5增

add(对象)
add_all([对象1，对象2， 对象3...])

db.session.add(对象)
db.session.commit()

9.6删

delete(对象)

db.session.delete(对象)
db.session.commit()

9.7改

db.session.add(对象)
db.session.commit()
注意: add(对象)这个方法可不写

9.8查

filter(模型名.字段 == 值), 注意filter中可以加多个条件，表示查询条件之间是且的操作。且filter()可以进行链式调用
filter_by(字段 = 值)
all(): 查询所有的数据，结果为列表
first(): 查询filter或filter_by结果中的第一个元素对象
get(id值): 获取主键所在行的对象，获取不到信息也不会报错
order_by(): 排序
order_by('-id'): 表示按照id的降序
order_by('id'): 表示按照id的升序
offset(数值1).limit(数值2): 表示跳过几个参数(数值1)，再截取几个参数(数值2)

9.9大小于

gt: 大于
ge:大于等于
lt: 小于
le:小于等于
>,<,<=,>=

9.10模糊

contains()： 包含
startswith(): 以什么开头
endswith(): 以什么结束
like(): 模糊查询
%: 表示匹配任何长度的数据
_: 匹配一位

9.11且或非

and_ : 且, 可不写，默认情况就是且的操作
not_ : 非
or_: 或

9.12paginate

has_next: 判断是否有下一页
next_num: 下一页页码
has_prev: 判断是否有上一页
prev_num: 上一页页码
page: 当前页
pages: 共多少页
iter_pages()： 循环脚码

10.flask-login

10.1注册

加密: generate_password_hash()
解密: check_password_hash()

10.2登录

安装: pip install flask-login
初始化: 
from flask_login import LoginManager
lm = LoginManager()
lm.init_app(flask对象)
登录用户: login_user(登录系统用户对象)
退出用户: logout_user()
登录校验: login_required
在模板中可使用{{ current_user.name }}渲染登录系统的用户名信息
重点: 用户的模型中需继承UserMixin，或者自定义相关的属性与方法
必须定义回调user_loader，用于返回登录系统的用户对象

11.模板

11.1父模板:

父模板负责挖坑{% bolock %}{% endblock %}, 挖的坑可以被子模板动态的填充内容

11.2子模板: 继承于父模板,并动态填充坑的内容

11.2-1继承:

{% extends '父模板' %}

11.2-2包含:

{% include '某个模板文件' %}

11.2-3获取坑之前填充的内容:

{{ super() }}

11.2-4加载静态文件

硬编码: 指定定义静态文件的路径
使用url_for进行解析: {{ url_for('static', filename='css/index.css') }}

11.2-5注解

<!--注解内容-->: 注意注解的内容中标签必须定义正确

注意: 和django的区别，没有comment标签

11.2-6标签

if标签: {% if  条件 == 值 %} {% endif %}
for标签: {% for i in 迭代对象 %} {% else %} {% endfor %}

11.2-7loop

获取从1开始增加的编号: {{ loop.index }}
获取从0开始增加的编号: {{ loop.index0 }}
倒序到1结束: {{ loop.revindex }}
倒序到0结束: {{ loop.revindex0 }}
判断循环第一次: {{ loop.first }}
判断循环最后一次: {{ loop.last }}
重点: jinja2模板引擎中没有ifequal标签

11.2-8macro宏定义

无参情况
{%  macro say() %}  {% endmacro %}

有参情况
{% macro say(name) %}  {{ name }} {% endmacro%}

引入
{% from 'functions.html' import say %}

使用: {{ say() }}

12.关联模型

12.1一对多

定义外键字段, db.ForeignKey('关联表名.id')
关联字段db.relationship('关联的模型名', backref='反向引用参数')
重点: relationship定义的字段，可以放在任何一个模型中

12.2多对多

定义外键字段是通过中间表，db.Table(表名, 字段)
关联字段db.relationship('关联的模型名', secondary='中间表', backref='反向引用参数')
重点: relationship定义的字段，可以放在任何一个模型中
